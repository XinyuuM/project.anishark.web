# 📝 自动化博客系统使用指南

## ✨ 重要更新

博客系统已升级为**自动侧边栏模式**，以后添加新文章**不需要手动更新配置文件**！

## 🎯 核心配置

在 `docs/.vitepress/config.mts` 中：

```typescript
sidebar: {
  // 自动侧边栏 - 从文件系统自动生成
  '/blog/': 'auto'
}
```

这行配置会让 VitePress 自动扫描 `docs/blog/` 目录下的所有文件，生成侧边栏导航。

## 📂 推荐的文件组织结构

```
docs/blog/
├── index.md                    # 博客首页（必需）
├── 2026/                       # 按年份组织
│   ├── 01-first-post.md       # 文章按数字前缀排序（推荐）
│   ├── 02-second-post.md
│   ├── 03-third-post.md
│   └── ...
└── 2025/                       # 往年文章
    └── ...
```

### 文件命名建议

使用数字前缀可以控制显示顺序：

```
01-文章标题.md        → 会显示在前面
02-另一篇文章.md
03-第三篇文章.md
```

## ✍️ 添加新文章的简单步骤

### 步骤 1: 创建文章文件

在对应的年份目录下创建新的 Markdown 文件：

```bash
# 示例：添加 2026 年 4 月的文章
cd docs/blog/2026
touch 04-2026-04-new-feature.md
```

### 步骤 2: 编写文章内容

```markdown
---
title: 新功能发布
description: 这是一篇关于新功能的文章
date: 2026-04-01
author: 你的名字
tags:
  - 功能更新
  - 新功能
---

# 新功能发布

这里是文章内容...
```

### 步骤 3: 更新博客首页（可选）

如果你想在新文章列表中展示这篇文章，可以编辑 `docs/blog/index.md`：

```markdown
## 📰 最新动态

### 2026年

#### [新功能发布](./2026/04-2026-04-new-feature)
*2026年4月1日*

文章简介...
```

### 步骤 4: 完成！

**就这么简单！** 不需要修改任何配置文件。

重启开发服务器后，新文章会自动出现在：
- ✅ 侧边栏导航
- ✅ 搜索结果
- ✅ 页面路由

## 🔄 自动工作原理

当你启动开发服务器时，VitePress 会：

1. 扫描 `docs/blog/` 目录
2. 读取所有 `.md` 文件
3. 根据文件系统结构生成侧边栏
4. 自动创建路由

因此，任何新增、删除、重命名文件的操作都会自动反映到网站上。

## 📋 文件 Front Matter（元数据）

建议每篇文章包含以下元数据：

```markdown
---
title: 文章标题              # 必需，显示在侧边栏和页面标题
description: 文章描述       # 可选，用于 SEO
date: 2026-04-01            # 可选，发布日期
author: 作者名              # 可选，作者信息
tags:                       # 可选，标签
  - 标签1
  - 标签2
---
```

## 🎨 自定义侧边栏显示

### 方法 1: 通过文件名控制顺序

使用数字前缀：
```
01-重要文章.md
02-普通文章.md
03-次要文章.md
```

### 方法 2: 通过文件夹分组

```
blog/
├── index.md
├── 重要/
│   ├── 01-文章.md
│   └── 02-文章.md
└── 普通/
    ├── 01-文章.md
    └── 02-文章.md
```

### 方法 3: 使用 `index.md` 自定义分组

在子目录中创建 `index.md` 来自定义分组标题：

```markdown
---
title: 2026年3月精选
---

本月份的精选文章...
```

## ⚠️ 注意事项

### 1. 不要删除 `index.md`

每个目录都应该有 `index.md`，作为该目录的首页。

### 2. 文件名建议使用英文或拼音

避免使用中文文件名，可能导致 URL 编码问题：

```
✅ good: 04-new-feature.md
✅ good: 04-xin-gong-neng.md
❌ 避免: 04-新功能.md
```

### 3. 避免特殊字符

文件名中避免使用空格和特殊字符：

```
✅ good: 04-new-feature.md
❌ 避免: 04 new feature!.md
```

## 🔍 故障排查

### 问题：新文章没有出现在侧边栏

**解决方案**：
1. 确认文件是否在正确的目录下（`docs/blog/` 或其子目录）
2. 检查文件扩展名是否为 `.md`
3. 重启开发服务器
4. 清除浏览器缓存

### 问题：侧边栏顺序不对

**解决方案**：
1. 使用数字前缀重命名文件
2. 调整文件夹的组织结构
